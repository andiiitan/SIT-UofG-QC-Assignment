# BB84 Quantum Key Distribution Simulation

This repository contains a Python-based simulation of the BB84 quantum cryptography protocol using IBM's Qiskit framework. It explores how quantum mechanics can be leveraged to establish a secure, shared cryptographic key between two parties, and demonstrates the physical impossibility of undetectable eavesdropping on a quantum channel.

## Repository Structure

* **`BB84-Plain.ipynb`**: Simulates the ideal execution of the BB84 protocol over a secure quantum channel. It demonstrates state preparation, transmission, measurement, and the final key sifting process to establish a matching secret key between Alice and Bob.
* **`BB84-Attacker.ipynb`**: Introduces an active eavesdropper ("Eve") utilizing an Intercept-and-Resend attack. It proves how the No-Cloning Theorem forces Eve to collapse the quantum states, ultimately allowing Alice and Bob to detect her presence.
  * *Bonus*: This notebook also includes a secondary demonstration of an advanced **Entanglement (CNOT) Attack**.
 
## Assignment Task Implementation

This project was done to directly fulfill the specific task requirements given:

* **"Implement a demonstration of the BB84 protocol, with and without an attacker."**
  * **Implementation:** The project is split into two distinct demonstrations: `BB84-Plain.ipynb` (notebook for simulation of the protocol without an attacker.) and `BB84-Attacker.ipynb` (notebook for simulation of the protocol with an attacker, to demonstrate that the attacker can be detected.).
* **"Implement it as a single sequential program — but make it clear which parts of the code correspond to each agent."**
  * **Implementation:** The notebooks execute sequentially to represent the linear flow of time. However, the code is clearly divided using comments and sections (e.g., Alice's Preparation, Eve's Interception, Bob's Measurement) to show exactly who is acting at what step.
* **"To generate the random choices... measure a suitable quantum state such as 1/√2(|0⟩ + |1⟩). Don’t use a standard Python random number generator."**
  * **Implementation:** The use of Python `random` module is strictly avoided. All random bits and basis choices are generated by applying a Hadamard gate to initialize a $|+\rangle$ superposition state, followed by a physical measurement to achieve true quantum randomness.
* **"The attacker should try to obtain information... for example by measuring some of the qubits. You can also try other kinds of attack."**
  * **Implementation:** The attacker notebook demonstrates the classic **Intercept-and-Resend Attack**, where Eve measures the qubits in transit. As a bonus, an advanced **Entanglement (CNOT) Attack** is also included to demonstrate "other kinds of attacks."
* **"The participants in the protocol should report an attack by using some threshold on the proportion of disrupted qubits."**
  * **Implementation:** To verify the integrity of the channel, Alice and Bob publicly sacrifice a sample subset of their sifted key. The system calculates the error rate in this sample; if the proportion of disrupted qubits exceeds a predefined security threshold (10%), the protocol correctly triggers an alert and reports the attack.
